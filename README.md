**ОПЕРАЦІЙНІ СИСТЕМИ РЕАЛЬНОГО ЧАСУ ТА 32-БІТНІ МК**
**Лабораторна робота 7.** Технологія використання операційної системи реального часу FreeRTOS при розробці ПЗ.

**Обладнання:** STM32F411 Discovery, CubeMX, Keil.

**Мета:** Ознайомитися з основами роботи ОС реального часу FreeRTOS, навчитися створювати потоки (tasks), керувати їх пріоритетами та часовими інтервалами за допомогою планувальника задач.

__________________________________________________________________________________________

**Завдання до лабораторної роботи (Варіант 4):**
1. Створити три потоки для керування світлодіодами:
   - **PD12**: частота 20 Гц, тривалість імпульсу 20 мс.
   - **PD13**: частота 10 Гц, тривалість імпульсу 40 мс.
   - **PD14**: частота 5 Гц, тривалість імпульсу 100 мс.
2. Реалізувати зміну тривалості імпульсу для **PD12** з 20 мс до 100 мс при натисканні кнопки **PA0**.

Цей проєкт демонструє переваги багатозадачності. Кожен світлодіод працює у власному ізольованому потоці, що дозволяє легко реалізувати складні часові діаграми без використання переривань таймерів або складних скінченних автоматів.

### 👨🏼‍💻 Функціонал:
* **Multitasking**: Паралельне виконання трьох незалежних завдань з різними часовими параметрами під керуванням FreeRTOS.
* **Preemptive Scheduling**: Використання витісняючого планувальника для перемикання контексту між задачами.
* **Dynamic Parameter Update**: Можливість змінювати логіку роботи одного з потоків у реальному часі через зовнішню подію (натискання кнопки).

### 📋 Ключові моменти реалізації:
1. **Task Creation**: Використання функції `xTaskCreate` (або `osThreadNew` у CMSIS-OS) для ініціалізації задач.
2. **vTaskDelay**: Забезпечення точних затримок, під час яких процесор перемикається на виконання інших потоків, замість марного очікування.
3. **Priority Management**: Встановлення однакових або різних пріоритетів для забезпечення стабільної роботи індикації.
